不知道 API Key 怎么获取?点这里看 1 分钟教程

① AI 模型 Key(审查用) — 默认用 DeepSeek:打开 platform.deepseek.com → 注册登录 → 左侧「API Keys」→ 点「创建 API key」→ 复制 sk- 开头的字符串,填到上方「AI 模型 API Key」。(需在 DeepSeek 账户里充值少量额度才能调用)

② 联网搜索 Key(找法规用) — 默认用 Tavily(注册即送免费额度):打开 app.tavily.com → 用邮箱或 Google 免费注册 → 登录后控制台首页就能看到 tvly- 开头的 API Key → 复制,填到「联网搜索 API Key」。

你填的所有 Key 只存在你自己的浏览器里,本站不上传、不保存。

LexScope 使用说明

用法规审视你的制度——上传内部制度,AI 对照内置权威法规库逐条审查,找出冲突、遗漏、越权、标准过宽,产出可留痕、可导出的合规审查报告。

适用场景

面向金融机构、企业(尤其企业集团财务公司)的内部制度合规审查。已内置 35 部权威法规、覆盖:公司治理 · 票据(承兑/贴现)· 信贷 · 关联交易 · 劳动用工 · 数据合规 · 反洗钱 · 财税等垂类。例如审查《审计委员会工作规则》《票据承兑/贴现业务操作细则》《信贷管理办法》等。

怎么用(三步)

  1. 上传制度 —— 拖拽或点击上传 Word / PDF / 图片,或直接粘贴文本。
  2. 分析 —— 点「开始审查」,AI 自动:判断该制度受哪些法律法规约束 → 在内置法规库中挑出相关条款 → 逐条全文体检 → 二次查漏复审。
  3. 审查报告 —— 每条发现一行三栏对照(受审文本原文 | 审查结论 | 外规依据+链接);逐条采纳/忽略盖章留痕;一键导出 PDF(文字可选、链接可点)。

名词解释

  • 受审文本:你上传、正在被审查的这份制度。
  • 外规依据:与受审文本比对的外部法律法规(及具体条款、原文链接)。
  • 置信度(高/中/低):AI 对该条结论的把握程度。
  • 待复核:置信度为中/低时标注,提示该条 AI 没十足把握,建议人工核对后再处置。
  • 链接存疑:该法条链接未通过校验,请点开人工核对原文。
  • 内置库 / 联网:该法规来自内置权威法规库 / 联网补充检索。
  • 审查依据:本次纳入对照的法规清单。
  • 审查覆盖:受审文本每一处用了什么法规核对、结论如何(合规/有风险/部分覆盖/未覆盖)。
  • 采纳 / 忽略:你对每条发现的处置决定,随报告 PDF 一起留痕。

隐私与边界

  • 自带 Key:DeepSeek 模型 Key 必填(建议用 deepseek-reasoner);联网搜索 Key 可选。密钥仅存于你本地浏览器,平台不上传、不留存。
  • 法规库:条文摘自官方来源并经交叉核验,个别标「待复核」;仅供审查参考,不构成法律意见,请以现行有效法规与人工复核为准。

常见错误对照(报错前缀【】即类型)

  • 【模型 Key 无效】:AI 模型 API Key 填错/过期。到 platform.deepseek.com 重新复制 sk- 开头的 Key;别把模型名(deepseek-reasoner)误填进 Key 框。
  • 【模型余额不足】:DeepSeek 账户需充值。
  • 【模型名称错误】:「模型名称」应填 deepseek-reasoner 或 deepseek-chat。
  • 【请求过于频繁】:触发限流,稍候重试。
  • 【网络错误】:断网或浏览器 CORS;DeepSeek 一般可直连,代理接口请配「转发前缀」。
  • 【返回为空 / 返回异常 / 解析失败】:模型没按要求输出,重试或改用 deepseek-reasoner。
  • 【未配置搜索 Key / 搜索 Key 无效 / 搜索网络错误】:联网搜索可选,留空仅用内置库即可。
